# 3 术语、定义和缩写

## 3.1术语和定义

围绕本文件的目的，使用下列术语和定义。

ISO和IEC为标准化工作维护的术语数据库可通过以下地址访问:

- IEC数字百科：<http://www.electropedia.org>
- ISO在线浏览平台：<http://www.iso.org/obp>
- IEEE在线标准词典：<http://ieeexplore.ieee.org/xpls/dictionary.jsp>

其他术语的定义一般可以在《ISO/IEC/IEEE 24765 系统和软件工程 词汇》(<http://www.computer.org/sevocab>)标准中找到。

### 3.1.13 概念操作 concept of operations
<!--ISO/IEC/IEEE 12207:2017# 新增术语-->

在后一种情况下，计划中的行动概念包括一系列将同时或相继进行的相互联系的行动。
这个概念的设计是为了给出组织运作的总体情况。
参见操作性操作(3.1.28)。

注2：
它提供了操作空间、系统功能、接口和操作环境的边界的基础。

### 3.1.14 关切 concern
<!--ISO/IEC/IEEE 12207:2017# 新增术语-->

系统中一个或多个利益相关者所关注利益。

注：
一个关切是在环境中影响系统的任何因素，包括发展、科技、商业、运作、组织、政治、经济、法律、法规、生态及社会影响。

[来源:ISO/IEC/IEEE 42010:2011]

### 3.1.15 配置项 configuration item
<!--GB/T 8566-2007# 3.6 配置项 configuration item-->

在配置管理中指代的指定硬件或软件的项或两者的聚合，在配置管理过程中作为独立实体处理。
<!--GB/T 8566-2007# 某个配置中的实体，它满足一项最终使用功能，并用给定的参考点上能够唯一地加以标识。-->

示例：
软件、固件、数据、硬件、人员、流程（例如为用户提供服务的流程）、程序(例如:操作接受和用户手册）、设备、服务、材料和自然存在的实体。

<!--ISO/IEC/IEEE 12207:2017# 新增术语-->

接受产品或服务的组织或个人。

示例：
消费者、客户、用户、收购者、购买者或购买者。

注：
客户可以是组织内部的，也可以是组织外部的。

注1到条目:每个启用系统都有自己的生命周期。本文件适用于每个启用系统，当其本身被视为系统的优先考虑对象时。

### 3.1.28 操作概念

一个组织关于一个系统或一组相关系统的操作或一系列操作的假设或意图的文字和图表的陈述

注1:操作概念的设计是为了从用户和操作人员的角度描述组织操作环境中使用一个或多个特定系统或一组相关系统的操作的总体情况。参见操作概念(3.1.13)。

### 3.1.29 操作符

执行系统操作的个人或组织

注1:
进入:操作员的角色和用户的角色可以同时或顺序地归属于同一个人或组织。
注2:
单个操作员与知识、技能和程序相结合可以被视为系统的一个组成部分。
进入注3:
操作员可以在操作的系统上或在操作的系统内执行操作，这取决于操作指令是否位于系统边界内。

### 3.1.30 组织

职责、权力和关系安排的人员和设施组
公司、公司、商号、企业、机构、慈善机构、独资商号、协会或其部分或组合。

注1:如果一个组织的一个已确定的部分(即使是单个个体)或一个已确定的组织组具有责任、权力和关系，则可视为一个组织。为某一特定目的而组织起来的一群人，如俱乐部、工会、公司或社团，是一个组织。

### 3.1.31 聚会，派对

签订协议的组织

注1:在本文件中，协议双方分别称为收购人和供应商。

### 3.1.32 问题

困难、不确定或以其他方式实现的不良事件、事件集、条件或需要调查和纠正行动的情况

### 3.1.33 过程

将输入转换为输出的一组相互关联或相互作用的活动

### 3.1.34 过程的结果

### 3.1.36 产品

过程的结果

注1:进入:有四种商定的通用产品类别:硬件(例如:，发动机机械部分);软件(如。、电脑程式程序，以及可能的有关文件和资料);服务(如。、运输);加工过的材料润滑剂)。硬件和加工材料通常是有形的产品，而软件或服务通常是无形的。

注1:项目有时被视为一个独特的过程，包括协调和受控的活动，并由本文件中定义的技术管理过程和技术过程中的活动组成。3.1.38

入口注1:关键质量特征通常包括与健康、安全、安全保证、可靠性、可用性和可支持性相关的特征。

### 3.1.45 资源

在过程执行过程中使用或消耗的资产

注1:资源包括可重复使用、可再生或可消费的资源。
例如不同的实体，如资金、人员、设施、资本设备、工具和公用事业，如电力、水、燃料和通信基础设施。

### 3.1.46 退休

操作和维护组织撤回积极支持，部分或全部替换为新系统，或安装升级的系统

### 3.1.47 风险

不确定性对目标的影响

注1:进入:一个效果是一个偏离预期-积极或消极。积极的影响也被称为机遇。

注2:目标可以有不同的方面(如财务、健康和安全，以及环境目标)，可以应用于不同的层次(如战略、组织范围、项目、产品和过程)。

进入注3:风险的特征通常是指潜在的事件和后果，或这些事件和后果的组合。进入注4:风险通常用事件的后果(包括环境的变化)和相关的发生可能性的组合来表示。
进入注意事项5:不确定性是与对事件、结果或可能性的理解或认识相关的信息不足的状态，甚至是部分信息不足的状态。

[来源:ISO指南73:2009，定义1.1]

注1:软件系统元素可以包括一个或多个软件单元、软件元素、硬件单元、硬件元素、服务以及其他系统元素和系统。

阶段

3.1.59
利益相关者
个人或组织在一个系统中或其拥有满足其需求和期望的特性中拥有权利、份额、要求或利益的个人或组织
例如最终用户、最终用户组织、支持者、开发人员、生产者、培训人员、维护人员、处置者、收购者、供应商组织和监管机构。
注1进入:一些利益相关者可能拥有相互对立或反对系统的利益。

供应商

注1至条目:供应商常用的其他术语为承包商、生产商、卖方或供应商。
注2:进入:收购人和供应商有时是同一组织的一部分。

系统
为达到一个或多个规定的目的而组织起来的相互作用的要素的组合

注1:
注2:在实践中，对其含义的解释常常通过使用联想名词来澄清，例如:、飞机系统或数据库管理系统。另外，单词“system”被上下文相关的同义词所代替，例如:尽管这可能掩盖了系统原理的观点。
